# Commit

Commit and push the updated design document. Use a descriptive commit message that summarizes what changed in the update (e.g. 'Expand auth design: fill in verification + alternatives sections').

**Outcomes:** committed, nothing-to-commit, other

**Transitions:**
- `committed` → **exit** (design-updated-and-committed)
- `nothing-to-commit` → **exit** (no-changes-to-commit)
- `other` → **exit** (user-provided-other)

**Model:** `haiku`
